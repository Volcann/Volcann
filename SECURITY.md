# Security Policy

## Supported Versions

The security team actively monitors and patch-validates only the latest version on the `main` branch.

| Version | Supported |
| ------- | --------- |
| main    | ✅ Yes    |
| < older | ❌ No     |

## Reporting a Vulnerability

If you discover any security-related issues, please do not open a public issue. Instead, report it privately:

1. Send an email to [mohsinmuhammad823@gmail.com](mailto:mohsinmuhammad823@gmail.com).
2. Detail the steps to reproduce the issue, along with any relevant workflow definitions or scripts.
3. We will acknowledge receipt of your vulnerability report within 48 hours and work with you to resolve it.

## Responsible Disclosure

We appreciate your research and cooperation. In accordance with standard industry practices, we ask that you:
* Allow us a reasonable amount of time to resolve the issue before making any information public.
* Avoid accessing or modifying data that does not belong to you.
* Act in good faith to preserve the integrity of user and repository data.
